## Installation

```bash
bun add @fern-ui/color-picker
```

Or copy `color-picker.tsx` and `color.ts` into your project — the component has
no runtime dependencies beyond React, and reads its colors from CSS custom
properties with literal fallbacks, so it themes where you have variables and
still renders where you do not.

## Usage

Pass `value` and `onChange` and you have the whole picker. Every capability is
on by default; you subtract with props rather than composing parts on.

## Live value vs settled value

`onChange` fires on every frame of a drag. `onChangeComplete` fires once, when
the interaction settles — pointer release, key press, or a committed text entry.

Drag the field below: the left swatch tracks every frame, the counter on the
right only moves when you let go. Use `onChange` for anything you are painting
and `onChangeComplete` for anything you would regret doing sixty times a
second — a network write, an undo entry, a re-render of the rest of the page.

<Preview>

```tsx
<ColorPicker
  value={color}
  onChange={setColor}                  // every frame — cheap work only
  onChangeComplete={(next) => {        // once, on settle
    save(next)
  }}
/>
```

## Choosing a color model

The select drives the row beneath it: `HEX` is one field, `RGBA` is four, `HSL`
is three. Switching reformats in place and never loses the color.

`format` sets which one it opens on. The value handed to your callbacks is in
that model too, and the copy button copies what the row displays — `#d97706` in
HEX, `rgb(217 119 6)` in RGBA.

<Preview>
  <PreviewDemo>
    <ColorPickerModel />
  </PreviewDemo>
  <PreviewCode>

```tsx
// Opens in RGBA. Users can still switch models unless
// you pass formatToggle={false}.
<ColorPicker format="rgb" value={color} onChange={setColor} />
```

  </PreviewCode>
</Preview>

Every channel field commits on blur or Enter and previews as you type, but only
from a complete, in-range value — a half-typed `#ff` is not a color, and `2555`
cannot be entered into a channel that stops at 255. Invalid input reverts rather
than clamping, so a typo reads as "that was rejected" instead of quietly
becoming a different color.

## Accessibility

- Arrow keys step 1, Shift+arrows step 10, and Home/End jump to the ends — on
  the saturation field and both sliders. The field is a primary control, not
  just a pointer target.
- Every channel input is a labelled `spinbutton` announcing its value in channel
  units — "Red, 77", not "77 percent" — and carries its own range.
- The model select points at the channel group it reshapes with `aria-controls`.
- Interactive targets are 40×40, clearing the WCAG 2.5.5 minimum.
- Settled values are announced through a polite live region. Intermediate drag
  frames deliberately are not, which would otherwise flood a screen reader with
  hundreds of updates per gesture.
- The field thumb flips between a light and dark ring based on the luminance
  beneath it, so it never disappears into the color it is selecting.
- A drag owns a single pointer id, so a second finger cannot hijack it
  mid-gesture.
- `prefers-reduced-motion` removes the thumb scale and the icon transitions.

The second argument to both callbacks is the full color — `hex`, `rgb`, `hsl`,
`hsv` and `alpha` — so you never have to convert by hand. The same conversion
helpers are importable on their own, without pulling in React:

```tsx
import { formatColor, parseHex, rgbToHsv } from "@fern-ui/color-picker/color"
```
