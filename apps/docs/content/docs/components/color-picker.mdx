## Anatomy

One component, no composition required. Each capability is a prop rather than a
subcomponent, so the default import is already the complete picker.

```tsx
<ColorPicker
  // Saturation and brightness field
  // Hue slider
  // Opacity slider        — alpha
  // Slider labels         — sliderLabels
  // Color model select    — formatToggle
  // Per-channel inputs
  // Current-color preview
  // Screen eyedropper     — eyedropper
  // Copy to clipboard     — copyable
/>
```

## Accessibility

- Every control is a labelled `role="slider"` carrying `aria-valuenow` and a
  human-readable `aria-valuetext`.
- Arrow keys nudge by 1%, Shift+arrows by 10%, and Home/End jump to the ends —
  matching native range inputs.
- Action buttons are 40×40, clearing the WCAG 2.5.5 target minimum.
- Each channel input is a labelled `spinbutton` announcing its value in channel
  units — "Red, 77", not "77 percent" — and carries its own range.
- The model select points at the channel group it reshapes with `aria-controls`.
- Settled values are announced through a polite live region; intermediate drag
  frames are deliberately not, which would otherwise flood a screen reader.
- The field thumb switches between a light and dark ring based on the luminance
  beneath it, so it stays visible across the whole gamut.
- `prefers-reduced-motion` removes the thumb scale and icon transitions.
- A drag owns a single pointer id, so a second finger cannot hijack it
  mid-gesture.

## API reference

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `value` | `string` | — | Controlled value. Accepts `#rgb`, `#rgba`, `#rrggbb`, or `#rrggbbaa`. |
| `defaultValue` | `string` | `"#3b82f6"` | Initial value when uncontrolled. |
| `onChange` | `(value, color) => void` | — | Fires on every change, including each frame of a drag. Use for live previews. |
| `onChangeComplete` | `(value, color) => void` | — | Fires once when an interaction settles. Use for network writes and undo history. |
| `format` | `"hex" \| "rgb" \| "hsl"` | `"hex"` | Starting color model. Users can change it at runtime unless `formatToggle` is false. |
| `formatToggle` | `boolean` | `true` | Let the user change the model. Disabling it locks the select. |
| `alpha` | `boolean` | `true` | Show the opacity slider and include alpha in output. |
| `sliderLabels` | `boolean` | `true` | Show the name and live readout above each slider. |
| `eyedropper` | `boolean` | `true` | Offer the native screen eyedropper where the browser supports it. |
| `copyable` | `boolean` | `true` | Show the copy-to-clipboard button. |
| `disabled` | `boolean` | `false` | Block interaction and dim the control. |
| `label` | `string` | `"Color picker"` | Accessible name for the group. |
